DNI electrónico

El Documento Nacional de Identidad electrónico (DNIe) es el documento de identidad electrónico que se expide en España desde marzo de 2006. Se trata de la evolución del anterior Documento Nacional de Identidad pensada para adaptar su uso a la sociedad de la información y para que sus portadores puedan darle uso para determinados servicios electrónicos.

LECTOR DE TARJETAS INTELIGENTES

Dispositivo físico que sirve para leer tarjetas con tecnología inalámbrica.

INTRODUCCIÓN

El programa diseñado tiene como función principal dar un servicio de autenticación básica usando el DNIe

Para la realización del programa se ha utilizado el entorno de trabajo NetBeans y se han utilzado tecno logías como php, java, html, css, javascript, etc

FUNCIONAMIENTO

El programa tiene dos partes, la aplicación cliente y la aplicación servidora.

En primer lugar el usuario deberá de conectar a su ordenador un lector de tarjetas inteligentes, ya sea mediante USB o si el lector viene integrado con el teclado.

Una vez conectado se deberá introducir el DNI electrónico en el lector para asi obtener la información del mismo

La aplicación cliente es la encargada de obtener la infromación del DNIe accediendo a los certificados del mismo, la aplicación cliente nos devolverá nuestro nombre y nuestro NIF. Una vez obtenidos dichos datos esta aplicación se conectará directamente con la aplicación servidora, la cual tiene una base de datos con una lista de usuarios que pueden acceder al servicio.

El acceso a la base de datos se realiza mediante dos parámetros

El valor del parámetro usuario sería una cadena de texto formada por las iniciales del nombre, primer apellido e inicial del segundo apellido (ejemplo: José Pérez lopez, sería el usuario jperezl.

El valor del parámetro clave será el NIF del usuario.

La aplicación cliente por si sola se encargará de realizar el login con la base de datos para comprobar si el usuario puede acceder al servicio.

La aplicación servidora como se ha dicho antes constará de una base de datos donde se almacenan todos los usuario aptos para acceder este servicio.

Tendremos también una página de inicio llamada index.html en la cual el usuario podrá acceder al servicio manualmente sin necesidad de usar el lector de tarjetas con el DNIe. Esta páginad e inicio tendrá un formulario donde se le pedirá al usuario un nombre de usuario y una contraseña. Recordamos que el nombre de usuario sera una cadena de texto formada por las iniciales del nombre, primer apellido e inicial del segundo apellido y la contraseña sera el NIF del usuario (letra incluida)

Una vez enviado el formulario se procederá a la autenticación del mismo y si ese usuario tiene acceso al servicio se mostrará la siguiente página
